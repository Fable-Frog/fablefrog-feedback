Use this repository to report bugs, request features, and share feedback. For open-ended discussion, head to [Discussions](../../discussions).

## How to Report an Issue

1. **Search first** — check [existing issues](../../issues) and [discussions](../../discussions) to avoid duplicates.
2. **Pick a template** — click [New Issue](../../issues/new/choose) and select the template that best fits:
   - **Bug Report** — something is broken
   - **Feature Request** — something you'd like to see
   - **UI/UX Feedback** — design and usability suggestions
   - **Playback Issue** — audio, streaming, or CarPlay problems
3. **Fill in the details** — the more context you provide, the faster we can act.

> **Server-side issues?** If your problem is with the Audiobookshelf server itself (not the Fable Frog app), please file it at [audiobookshelf/audiobookshelf](https://github.com/audiobookshelf/audiobookshelf/issues).

## How Triage Works

We use a hybrid system of **community reactions** and **maintainer labels** to prioritize work.

### Community Reactions

**Your upvotes directly influence what gets built next.** Add a :thumbsup: reaction to any issue you care about. We regularly review issues sorted by reaction count to understand what the community wants most.

- :thumbsup: = "I want this too" / "I'm affected by this bug"
- Don't comment just "+1" — use the reaction button instead to keep threads clean.

### Maintainer Status Emoji

We use emoji reactions on the **issue itself** (not comments) to communicate status:

| Emoji | Meaning |
|---|---|
| :eyes: | Under review |
| :white_check_mark: | Confirmed / accepted |
| :rocket: | Planned for development |
| :tada: | Shipped |

### Labels

Labels provide official categorization:

- **Priority**: `priority/critical`, `priority/high`, `priority/medium`, `priority/low`
- **Status**: `needs-triage`, `confirmed`, `in-progress`, `planned`, `shipped`, `needs-info`
- **Type**: `bug`, `enhancement`, `ui/ux`, `playback`
- **Area**: `area/library`, `area/player`, `area/downloads`, `area/carplay`, `area/sync`, `area/auth`, `area/widgets`, `area/playlists`

## Stale Issues

To keep the tracker clean:
- Issues with no activity for **60 days** are marked `stale`.
- After **14 more days** of inactivity, they're automatically closed.
- Issues labeled `priority/critical`, `priority/high`, `planned`, or `in-progress` are exempt.
- Closed issues can always be reopened if the problem resurfaces.
- Issues labeled `needs-info` with no author response for **14 days** are closed (reopen anytime with the missing info).
